LITMUS

Autonomous data exploration agent. Forms hypotheses, runs experiments, tries to kill its own findings. Reports only what survives.

The Operator Agentic Autonomy Tool Use State Management Multimodal Input
The Problem

Existing tools answer questions. This one finds them.

Pandas Profiling produces summaries you already expected. LLM chat answers the question you typed. AutoML optimizes a target you defined. All reactive. All waiting for your input.

LITMUS runs autonomously. It profiles your data, generates hypotheses ranked by information gain, executes statistical tests, attempts to debunk its own results, and surfaces only findings that pass a 5-stage validation gauntlet. You upload a dataset. It tells you what you missed.

Architecture

The Discovery Loop

1
PROFILER
Schema inference, distributions, correlations, outlier census, missing data patterns
2
HYPOTHESIZER
Generates ranked hypotheses by expected information gain. Decides where to look next.
3
EXPERIMENTER
Writes + executes statistical tests in sandbox. scipy, statsmodels, sklearn. Generates live plots.
4
SKEPTIC
Challenges its own findings through 5 validation checks. Failed results are archived, not discarded.
5
NARRATOR
Produces narrated findings with cited evidence, confidence grades, and embedded plots.
Results from step 3 update belief state and feed back into step 2. Each experiment informs the next.
Two Modes

Discovery Mode + Directed Testing Mode

Discovery Mode

"Here's my data. What's in it?"

Autonomous exploration. LITMUS generates its own hypotheses, runs experiments, validates findings. Zero guidance needed. For researchers, analysts, or domain experts sitting on data they haven't fully examined.

Directed Testing Mode

"Here's my hypothesis. Test it rigorously."

User describes a hypothesis in plain English. LITMUS determines the appropriate test, checks assumptions, falls back to non-parametric alternatives when assumptions fail, runs power analysis, computes effect sizes, and produces a plain English interpretation.

Directed mode handles significance tests, A/B tests, multi-variant experiments, and segment breakdowns. The user who knows what to test but not how gets the same statistical rigor as someone writing scipy by hand.

What Directed Mode automates:
TEST SELECTION
Picks t-test, chi-square, Mann-Whitney, ANOVA, etc. based on data types, sample size, and distribution shape
ASSUMPTION CHECKING
Shapiro-Wilk for normality, Levene's for variance homogeneity. Auto-falls back to non-parametric when violated.
POWER ANALYSIS
Flags underpowered experiments before wasting time. Reports minimum sample size needed for the target effect.
A/B TEST SUITE
Sequential testing, multiple variants, guardrail metrics, segment breakdowns. Full experiment report with effect sizes.
Example: Directed Mode Output Directed Test

User input: "Did the new checkout flow improve conversion rates?"

LITMUS: "Group B converted 12% higher than Group A (18.3% vs 16.4%). Normality check passed (Shapiro-Wilk p = 0.34). Applied Welch's t-test: t(1847) = 3.12, p = 0.003, Cohen's d = 0.48 (medium effect). Power = 0.91 at current sample size. Result held across all 3 user segments (mobile, desktop, tablet). No guardrail metric regressions detected."

Test: Welch's t-test p-value: 0.003 Effect: medium (d=0.48) Power: 0.91
Validation

The Skeptic Gauntlet

Every finding passes through five validation checks before being reported. Findings that fail validation are demoted and archived with full reasoning, not deleted. The entire experimental trail is preserved.

CHECK 1
Multiple Testing
Benjamini-Hochberg FDR
CHECK 2
Confounder Scan
Partial correlations
CHECK 3
Simpson's Paradox
Stratified analysis
CHECK 4
Holdout Replication
70/30 split validation
CHECK 5
Effect Size Filter
Cohen's d > 0.3

Confidence grades: A = survived all 5 / B = survived 4 / C = failed validation (archived)

Ranking

The Surprise Score

Statistical significance alone is not enough. LITMUS ranks by how unexpected a finding is relative to the data's baseline.

discovery_score = surprise * significance * effect_size
surprise = KL divergence from expected distribution  |  significance = -log10(adjusted p-value)  |  effect = |Cohen's d| or |r|

Obvious confirmations get deprioritized even at p < 0.001. Only findings that are statistically robust AND genuinely unexpected surface to the top.

Track Fit

Operator Rubric Alignment

Agentic Autonomy

Zero human input after upload. Agent decides what to test, when to go deeper, when to prune. Explore/exploit tradeoff drives the loop.

Tool Use & Integration

Dynamic tool calls: code executor, statistical libraries, visualization generator, Google Sheets API. Agent selects tools based on hypothesis type.

Multimodal & Unstructured

Accepts CSV, JSON, Excel, pasted tables via Claude Vision. Handles messy headers, mixed types, encoding issues. Messy in, structured out.

State & Context Mgmt

Belief state persists across rounds. Hypothesis tree grows and prunes. Experiment N is shaped by everything before it.
